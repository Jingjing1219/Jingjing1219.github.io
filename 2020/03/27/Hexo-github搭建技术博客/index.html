一些基本概念介绍

首先从What,Why这两个方面来介绍一下我们的文章主题。
（官网介绍）Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
Hexo是一款基于Node.js的静态博客框架，依赖少易于安装使用，可以方便的生成静态网页托管在GitHub和Heroku上，是搭建博客的首选框架。（摘自https://zhuanlan.zhihu.com/p/26625249）
为什么要将博客部署到github上？ 首先免费，其次省心，最后可以学习使用 GitHub。（参考链接 https://io-oi.me/tech/hexo-next-optimization/）

githubSSH配置

前面 git for window 安装完成后,打开命令控制台，输入 git，出现以下信息表示安装成功

然后打开安装路径下的 Git Bash.exe

出现以下界面，而后输入命令：ssh-keygen -t rsa -C “你的邮箱”。（相当于本地生成一个链接你账号的密匙）

然后打开github，选择setting

找到SSH and GPG keys，点击 New ssh key

Title 随便写，Key 里面的内容，是将文件id_rsa.pub里面的内容复制过来的。（注意id_rsa.pub的位置，在你刚刚 输入命令：ssh-keygen -t rsa -C “你的邮箱” 这一步的时候，界面会出现你的.ssh文件的位置）

至此githubSSH配置完成

hexo本地使用

基本介绍

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。（官网介绍）

基于在hexo通过markdown编写的文章，然后hexo帮我们生成静态的html页面，然后，将生成的html上传到我们的服务器。简而言之：hexo是个静态页面生成、上传的工具。参考博客1

使用过程

百度搜索HEXO，点开hexo官方文档

在D盘新建一个blog的文件夹（ 这个文件夹作为你的博客的主目录，然后使用CMD命令行进入到这个目录下执行初始化命令：hexo init）。
打开控制命令台，到D文件夹下的blog文件夹里，然后输入安装命令，等待安装完成。

然后初始化博客（这个过程比较长）: hexo init blog

上图红方框表示初始化完成了，但是黄方框代表什么？我看别的教程上好像没有这个现象，但是好像对后面的过程没有什么影响。
接下来，执行npm install命令。也出现了上面黄色框标出来的问题。

下面输入hexo s -p 5555，也可以直接hexo s。 (s表示start)

进入上面提示的网页，就可以打开我们的本地博客了。

hexo几个基本指令总结

Hexo指令总结：
Hexo clean 清理一下；
Hexo g: 生成（generate）；
Hexo s -p 5555: 发布到localhost:5555的博客网站（hexo s 启动预览（start））
Hexo d: 提交到github上；

博客的一些设置

  • 增加侧栏菜单条目
    默认的侧栏菜单条目有：首页、归档、标签、关于、搜索等。如果你想要增加其他的菜单条目，修改主题配置文件_config.yml里的Menu Settings中的menu和menu_icons两个地方。

  • 插入图片之图床法
    这时考虑把博文里的图片上传到某一网站，然后获得外部链接，使用Markdown语法， 完成图片的插入，这种网站就被成为图床。常见的简易的图床网站有：贴图库图片外链 国内算比较好的图床我认为有两个：新浪微博和 七牛云 ，七牛云的使用方法可以参看其他文章。图床最重要的就是稳定速度快，所以在挑选图床的时候一定要仔细。参考博客
